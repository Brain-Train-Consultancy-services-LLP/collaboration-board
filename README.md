# Collaboration Board

Team Formation and Open Collaboration Hub

This repository serves as the central coordination space for forming teams and initiating collaborative projects.

Before starting any project, please read the guidelines below carefully.

---

## Purpose of This Repository

This space is used exclusively for:
- Finding teammates
- Looking for specific roles (ML, Backend, Documentation, Product, etc.)
- Inviting international collaborators
- Open discussions around team formation

All collaboration happens through GitHub Discussions.
Teams form voluntarily. No tasks are assigned here.

This is not a task board or assignment system.
Leadership and ownership emerge organically

---

## How to Get Started

If you are interested in a problem prompt and want to collaborate:
1. Open a discussion in this repository
2. Clearly state:
   - What problem you want to work on
   - What kind of collaborators you are looking for
3. What kind of collaborators you are looking for
4. Create a separate project repository for your work

---

## Non-Negotiable Participation Rules

- Only team-based projects are eligible.
- Solo repositories will not be reviewed or evaluated.
- Each team must have a minimum of 5 active contributors.

These rules exist to encourage collaboration, leadership, and real-world teamwork.

---

## Project Repository Rules (Mandatory)

Each team must create its **own separate project repository**.

Repositories that do not meet the requirements below will not be observed or evaluated.

---

## Team Visibility(Optional)

Teams may optionally register themselves by creating a GitHub Issue in this repository.

Required information:
- Project repository link
- Team size
- Countries represented
- Team Lead GitHub ID
- Applied Smart Intern present: Yes / No

This registration provides visibility only.
No approval, control, or ranking is exercised.

---

## Standard README Format (Mandatory)

Each project repository must include a `README.md` file using the format below.

# Project Name

## Problem Statement
Describe the problem the project aims to address.

## Why This Problem Matters
Explain the real-world relevance and impact.

## Team Members
- Name 1
- Name 2
- Name 3
- Name 4
- Name 5

## Team Lead
- Name

## Current Status
Mention the current stage of the project.
Examples:
- Research ongoing
- Prototype ready
- Testing in progress

## How We Are Working
- How tasks are divided among team members
- Whether weekly updates or discussions are happening

## Tech Stack (Optional)
List the tools or technologies being used, if any.

## Contact
Provide GitHub usernames or LinkedIn profile links.


---

## Repository Eligibility Rules (Mandatory)

To be considered a valid team project, **each repository must include all of the following files**:

### 1. TEAM.md
This file must clearly mention:
- Team Lead
- Team Members
- Countries
- Time Zones
- Roles and responsibilities

### 2. CONTRIBUTION.md
This file must describe:
- How team members contribute
- How work is reviewed
- How collaboration is managed

### 3. README.md
This must include:
- Project overview
- Problem being solved
- Team structure
- Collaboration workflow

Repositories missing any required file will not be observed or evaluated.
